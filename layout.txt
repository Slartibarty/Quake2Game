cgame:

The client game DLL

common:

Code, headers and misc sharable with everything, even utilities
ideally the cgame and game shouldn't be aware of the file formats, that should be handled by the engine

engine:

The glue that binds the client and server together, as well as the two games handles low-level functionality
and OS-specific implementations of core stuff (input, rendering, sound, etc)

game:

The server game DLL

thirdparty:

Thirdparty supporting libraries

utils:

Standalone development tools

root:

contains premake script, regen scripts, maintenance stuff
